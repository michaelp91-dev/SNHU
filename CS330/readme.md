
CS-330 Portfolio Reflection
1. How do I approach designing software?
When approaching software design, I focus on breaking down the problem into clear, manageable components before writing any code. For the final project, I started by analyzing the 2D image and identifying which basic 3D shapes could best represent each object (plane for the desk, box + plane for the tablet, cylinder + torus for the mug). This helped me plan the scene layout and transformations early.
A new skill I developed was thinking about modularity from the start. Instead of putting everything in one big function, I created dedicated functions like LoadSceneTextures(), DefineObjectMaterials(), and SetupSceneLights(). This made the code much easier to debug and extend. I also learned to separate concerns — keeping rendering logic, collision detection, and input handling in their own logical sections.
In the future, I will continue using this structured design approach, especially when working on larger AI or simulation projects where clean architecture makes a big difference.
2. How do I approach developing programs?
My development process became much more iterative throughout this course. In the early milestones, I would often try to implement everything at once and then spend hours debugging. By the later milestones and final project, I adopted a “build small, test often” approach — adding one feature (like multi-hit bricks or proper lighting), testing it thoroughly, then moving to the next.
One of the biggest improvements was learning to use modular functions and reusable code. Instead of repeating transformation and material code for every object, I created helper functions that could be called anywhere. This not only saved time but also made the code much cleaner and easier to maintain.
I also became better at debugging complex issues, such as the lighting always following the camera (view-space vs world-space problem). Breaking the problem down and testing small changes helped me solve it more efficiently than I could have earlier in the course.
3. How can computer science help me in reaching my goals?
Computational graphics and visualization have given me practical skills that directly support both my educational and professional goals. In my AI engineering program, I now have a much stronger understanding of how 3D environments are built and rendered — knowledge that will be valuable when working on AI-driven simulations, medical visualizations, or game development tools.
Professionally, these skills have shown me how important clean code architecture and iterative development are in real-world graphics programming. Being able to create a fully functional 3D scene with proper lighting, textures, camera controls, and collision systems has given me confidence that I can contribute to projects involving real-time rendering or interactive applications.
Overall, this course has strengthened both my technical abilities and my problem-solving mindset, which I know will serve me well as I continue my education and eventually enter the tech industry.
